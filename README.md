ESP32 Vision-Controlled Robot
=============================
A two-MCU robot system that uses computer vision to track a person and respond to hand gestures. A laptop runs YOLOv8 person detection and MediaPipe hand-landmark classification (along with a custom gesture classification model), then streams motor commands over UDP to an ESP32-WROOM-32D. An ESP32-CAM provides the live MJPEG video feed.


PROJECT STRUCTURE
-----------------
platformio.ini          - PlatformIO build config (two environments: esp32 and esp32cam)
src/                    - ESP32-WROOM-32D firmware (ESP-IDF)
  useful_stuff.h        - Shared library imports and typedefs
  imu.h                 - MPU-6050 I2C interface declarations
  imu.c                 - MPU-6050 I2C interface
  motors.h              - Motor driver interface declarations
  motors.c              - Motor driver interface
  connections.h         - WiFi and UDP listener declarations
  connections.c         - WiFi and UDP listener
  main.c                - Main logic for ESP32-WROOM-32D
cam_firmware/           - ESP32-CAM firmware (Arduino)
  board_config.h        - Selects the AI-Thinker camera model
  camera_pins.h         - GPIO pin definitions for all supported ESP32-CAM board models
  app_httpd.cpp         - HTTP stream server
  camera.cpp            - Camera init, sensor tuning, and WiFi connection
cv_tasks/               - Python computer vision pipeline
  collect_data.py       - Captures webcam images to dataset/ for each gesture class
  train_gesture_clf.py  - Trains gesture classifier and saves gesture_clf.pkl
  main.py               - Reads ESP32-CAM stream and sends UDP commands based on inference data


DEPENDENCIES
------------
Install PlatformIO. All libraries are resolved automatically on build.

Python 3.9+:
  pip install opencv-python mediapipe ultralytics scikit-learn numpy


MODELS AND DATASETS
-------------------
yolov8n.pt
  Auto-downloaded by the ultralytics library on first run

hand_landmarker.task
  Must be downloaded manually into cv_tasks/ before running cv_controller.py or
  gesture_trainer.py:
    wget https://storage.googleapis.com/mediapipe-models/hand_landmarker/hand_landmarker/float16/1/hand_landmarker.task

gesture_clf.pkl
  Generated by running gesture_trainer.py after collecting a dataset. My given version classifies between the following gestures: palm, back of hand, fist facing camera, and pointing down with index finger.

dataset/
  Collected locally using gesture_collector.py. Only needed if training for own gestures.


HOW TO RUN
----------
1. Update WiFi credentials in cam_firmware/camera.cpp (WIFI_SSID,
   WIFI_PASSWORD) and src/connections.h (same defines) to match your network.

2. Update the port numbers in the platformio.ini file for the ESP32-WROOM-32D and ESP32-CAM
   with the ports you're using for each.

3. Flash ESP32-CAM:
     pio run -e esp32cam --target upload
   Note the IP address printed to serial monitor and set ESP32_CAM_IP in main.py.

4. Flash ESP32-WROOM-32D:
     pio run -e esp32 --target upload
   Note the IP address printed to serial monitor and set UDP_IP in main.py.

5. Run live inference:
     python main.py
   Must be in the cv_tasks directory when running above command to ensure it can detect models.
   Press q to quit.
